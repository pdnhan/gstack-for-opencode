Investigate the current bug or unexpected behavior using a root-cause-first debugging workflow.

Requirements:

- do not jump straight to a fix
- gather evidence from symptoms, relevant code paths, and recent changes
- state a clear root-cause hypothesis before implementation
- if a fix is made, keep it minimal and specify how to verify it
- include a regression test when the bug is fixable in code
