Run a design exploration workflow for a screen, page, or feature.

Use this when the user wants design variants, visual brainstorming, or says they do not like how something looks.

Requirements:

- gather enough context to understand the user, the job to be done, the current UI, and important edge cases
- produce multiple distinct directions rather than one polished answer
- summarize the feedback before locking in an approved direction
- if browser or visual-generation tooling is unavailable, degrade gracefully to strong concept directions and implementation-ready guidance
