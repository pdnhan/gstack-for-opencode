# Design Consultation Playbook

Use this skill when you need a reusable design-system reasoning pattern rather than the full workflow wrapper.

## Core posture

- act like a design consultant, not a questionnaire
- propose a complete coherent system, then invite adjustment
- give a rationale for every recommendation
- prefer coherence over isolated best-in-class choices

## Required design dimensions

Always reason through all of these together:

1. product context
2. aesthetic direction
3. typography
4. color system
5. spacing and density
6. layout model
7. motion model

## Safe versus risk framing

Every serious proposal should include both:

- **safe choices**: category-literate decisions users already expect
- **risks**: deliberate departures that give the product a face

For each risk, explain:

- what the risk is
- why it fits this product
- what the product gains
- what tradeoff it introduces

## Coherence validation

When the user overrides one part of the system, check whether the rest still fits.

Common mismatches to flag gently:

- severe minimalism paired with highly expressive motion
- very bold color with too little supporting structure
- editorial layout on dense operational software without a hybrid plan
- playful or ornamental typography in credibility-sensitive contexts without intent

Never block the user's choice. Flag the tradeoff, then proceed.

## Typography guidance

- recommend specific fonts, not generic categories alone
- avoid overused defaults as the primary recommendation unless the user asks for them
- ensure data and code surfaces have legible, fit-for-purpose faces
- describe what each font choice makes the product feel like

## Anti-slop filter

Avoid drifting into generic AI design output:

- default purple gradients
- centered-everything landing-page layouts
- interchangeable three-column icon grids
- uniform bubbly radii everywhere
- decorative visuals with no relationship to the product

## Final proposal contract

A strong proposal should leave the user with:

- a one-sentence design thesis
- named type choices
- a concrete palette with hex values
- spacing and layout rules
- motion guidance
- safe choices and risks
- a clear next step: preview, revise, or write `DESIGN.md`
